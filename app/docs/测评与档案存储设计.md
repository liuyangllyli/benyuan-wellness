# 测评与档案存储设计（GPT + Claude 综合定稿）

**依据**：`docs/测评与数据架构说明书_供AI协作用.md` 中「三、需要其他 AI 协助回答的问题」；综合 GPT 与 Claude Code 的答复后定稿，供实现与后续云端同步使用。

---

## 一、总设计原则

- **Profile** = 当前有效档案（current snapshot）：首页/推荐/各页即时读取。
- **History** = 不可变的测评提交记录（每次提交一条）：用于复现当次结果、回溯、未来云端同步。
- **写顺序**（采纳 Claude）：先更新 profile 对应块，再 append 一条 history。profile 为「当前态」，history 为「日志」；history 的 payload 存当次完整数据，可复现。
- **分块**：基本信息、体质、身体测试三块独立存储与更新，每块带 `updatedAt`；支持单块更新与 per-block last-write-wins（未来多端同步）。

## 二、当前档案 benyuan_profile 结构（v1）

```ts
// schemaVersion：便于未来迁移
// 三块 + 全局时间戳 + 状态 + 派生（可选）
interface BenyuanProfile {
  schemaVersion?: number  // 1

  basicInfo?: {
    specialConditions?: string[]  // 题1 多选
    ageRange?: string             // 题2 年龄区间
    gender?: string               // 题3 性别
    occupation?: string           // 职业（可选）
    updatedAt?: number
  }

  constitution?: {
    answers: (number | string)[]
    baseScores?: number[]
    actualScores?: number[]
    dimensionScores: Record<string, number>
    dimensionConvertedScores: Record<string, number>
    type: string
    tendency?: string[]
    pianpoYes?: string[]
    summary?: string
    updatedAt?: number
  }

  bodyTest?: {
    q4?: number
    q5?: number
    q6?: number[]
    q7?: number
    q8?: number
    q9?: number
    q10?: number
    q11?: number
    q12?: number
    q13?: number[]
    q14?: number[]
    q15?: number
    q16?: number
    q17?: number
    q18?: number
    q19?: number
    score4_12?: number
    evaluation?: string
    overtrainingWarning?: boolean
    exerciseMethods?: string[]   // ['活身法','动身法','合身法']
    updatedAt?: number
  }

  timestamp?: number
  lastFullAssessmentAt?: number

  status?: {
    hasBasicInfo: boolean
    hasConstitution: boolean
    hasBodyTest: boolean
  }

  derived?: {
    specialSituationSelected?: boolean   // 题1 是否选过特殊情况
    minorOrElderNeedCompanion?: boolean  // 年龄<12 或 >70
  }
}
```

- 读：首页/炼体/食养/档案页只读 `benyuan_profile`。
- 写：提交某块时 merge 该块到 profile，并更新该块 `updatedAt`、全局 `timestamp`；若为完整三步则更新 `lastFullAssessmentAt` 与 `status`。

## 三、测评历史 benyuan_assessment_history 结构

```ts
type AssessmentRecordType = 'basic_info' | 'constitution' | 'body_test'

interface AssessmentRecord {
  id: string
  timestamp: number
  type: AssessmentRecordType
  payload: {
    basicInfo?: BenyuanProfile['basicInfo']
    constitution?: BenyuanProfile['constitution']
    bodyTest?: BenyuanProfile['bodyTest']
  }
  logicVersion?: string
}
```

- 每次只提交一块：`type` 为 basic_info / constitution / body_test 之一，`payload` 仅含对应块，保证可复现当次结果。
- 历史列表按 `timestamp` 倒序；详情页用 `history[i].payload` 渲染，不依赖当前 profile。
- 本地 cap 50 条；后续可做按 type 筛选与分页。

---

## 四、写顺序（提交某块时）

1. 根据当次答案与算分逻辑，得到该块完整数据（含 updatedAt = Date.now()）。
2. `saveProfile({ [block]: data })`：merge 进当前 profile，更新 timestamp。
3. `appendAssessmentHistory({ id, timestamp, type, payload: { [block]: data }, logicVersion })`。

## 五、页面与数据来源

| 页面           | 数据来源           | 说明 |
|----------------|--------------------|------|
| 测评结果页     | 内存 + profile     | 提交后展示当次结果，并写入 profile |
| 我的测评档案   | profile 三块        | 三块摘要 + 更新 + 查看历史 |
| 历史记录列表   | history 数组        | 时间 + 类型 + 摘要 |
| 历史详情       | history[i].payload  | 按 payload 复现当次结果 |
| 首页/炼体/食养 | profile            | onShow 读 profile 对应字段 |

## 六、路由与入口（实现时对齐）

- 我的测评档案：`/pages/assessment/archive`（或 `/pages/profile/my-assessment`），入口：首页卡片 / 观己默认进此处（已测评显示摘要，未测评显示「开始测评」）。
- 历史列表：`/pages/assessment/history`，入口：档案页「查看历史记录」。
- 历史详情：`/pages/assessment/history-detail?id=xxx`，入口：点击历史某条。
- 测评页：`/pages/assessment/assessment`，支持 `mode=basic_info_only` | `constitution_only` | `body_test_only` 单块更新。

---

## 七、旧 profile 迁移

- 若读取到旧版 profile（无 `schemaVersion`，有 `constitutionAnswers`、`constitutionType` 等顶层字段），自动迁移为 v1：写入 `constitution` 块，并设置 `schemaVersion: 1`、`status`。
- 迁移在 getProfile 时惰性执行并写回，保证后续读写均为 v1 结构。
