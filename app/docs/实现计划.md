## 三、建议实现顺序（在 Cursor 中逐页打磨）

1. **Onboarding** → 三步引导、写 `benyuan_onboarded`、出口「了解自己」/「先逛一逛」

每页实现时请对照：`docs/architecture.md`、`docs/key-code-index.md`、`docs/reference/每页功能与逻辑要点.md`。

## 四、tabBar 隐藏说明

Lovable 在 `/onboarding`、`/calm` 隐藏底部导航。uni-app 原生 tabBar 无法按页面动态隐藏，可选方案：

- **方案 A**：onboarding、calm 使用自定义全屏页（已设 `navigationStyle: "custom"`），用户在这些页看不到 tabBar（因为不是 tab 页，不会带 tabBar）。
- **方案 B**：若需与其它非 tab 页一致地「隐藏 tabBar」，需使用**自定义 tabBar** 组件，根据当前页面 path 决定是否渲染。

当前已采用方案 A：onboarding、calm 为普通非 tab 页，本身就不会出现原生 tabBar。

## 五、lib 与组件

- **lib**：在 `app/src/lib/` 下实现与 Lovable 等价的工具（如 `profile-utils`、`time-utils`、`solar-terms`、`api-contracts` 的 stub），键名与文档一致。
- **组件**：按 key-code-index 的页面级子组件归属，在对应页面中引入或逐步抽成 `components/` 下组件。

细节在 Cursor 中按需补充即可。
