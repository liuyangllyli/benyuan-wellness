# 手机上看 & 分享给伙伴测试

## 一、在手机上打开（同一 WiFi）

1. 在电脑上进入应用目录并启动 H5：
   ```bash
   cd app
   npm run dev:h5
   ```
2. 终端里会显示类似：
   - `Local:   http://localhost:5179/`
   - `Network: http://192.168.x.x:5179/`
3. **手机连与电脑相同的 WiFi**，在手机浏览器地址栏输入 **Network 里的地址**（如 `http://192.168.x.x:5179`），即可打开测评页面。
4. 若打不开：检查电脑防火墙是否放行 5179 端口，或暂时关闭防火墙试一下。

## 二、分享给合作伙伴测试

### 方式 A：伙伴和你在同一办公室 / 同一 WiFi

- 把上面的 **Network 地址**（如 `http://192.168.1.100:5179`）发给他们。
- 对方手机/电脑连同一 WiFi，浏览器打开该地址即可。
- 注意：你的电脑需一直开着并保持 `npm run dev:h5` 在运行。

### 方式 B：伙伴在异地（需要公网可访问的链接）

任选其一即可：

1. **内网穿透（开发机临时分享）**  
   用 [ngrok](https://ngrok.com/)、[localtunnel](https://localtunnel.github.io/www/) 等，把本机 5179 映射成一个公网 URL，把该 URL 发给对方。  
   示例（localtunnel，无需注册）：
   ```bash
   npx localtunnel --port 5179
   ```
   会得到一个类似 `https://xxx.loca.lt` 的链接，发给他们即可。你电脑上的 `npm run dev:h5` 需同时运行。

2. **部署到线上再分享（更稳定）**  
   - 在 `app` 目录执行：`npm run build:h5`，会在 `app/dist/build/h5` 生成静态文件。  
   - 把该目录部署到任意支持静态托管的服务（如 Vercel、Netlify、阿里云 OSS、自己的服务器等）。  
   - 把部署后的访问地址（如 `https://your-app.vercel.app`）发给伙伴，他们随时可测，不依赖你电脑开机。

## 三、数据说明

- 当前为**本地存储**：每个设备/浏览器各自存一份，手机和电脑、你与伙伴之间的数据**不互通**。  
- 换设备或清除浏览器数据后，之前的测评记录会丢失，这是预期行为，后续接后端后可同步。
