## 一、当前已实现

### 1. 测评与档案（观己）

- **三步流程**：Step1 基本信息（题1–3 + 职业）→ Step2 体质 67 题 → Step3 身体测试（题4–19）；步骤条、进度条。
- **单块编辑**：从观己（档案页）进入「编辑」任一块（基本信息 / 体质 / 身体测试），保存后仅更新该块；身体测试编辑时「更新最近一条历史」不新增记录。
- **弹窗**：进入身体测试前若存在「特殊情况」或「年龄陪同」，弹窗提示；点「暂不填写」会正确返回（body_test_only 时 `switchTab` 回观己）。
- **体质 37/38 题**：按性别灰调；未选性别时显示「两道题均需作答」提示。
- **档案页**：完整测评摘要（三块）、测评历史（默认收起，点「查看」展开）、历史详情页（可「返回」「返回档案」）；时间格式统一 `YYYY-MM-DD HH:mm`；加载失败有 toast；三块都空时有「约 9 分钟完成三步」提示。
- **文案**：测评与档案相关文案集中在 `app/src/lib/assessment-copy.ts`；时间格式化在 `app/src/lib/format.ts`。

### 2. 身体测试逻辑

- **算分与评价**：`app/src/lib/body-test-scoring.ts`（4–12 总分、评价分段、炼体法触发）。
- **过量提醒**：题7 低强度 120 分钟以上、题8 中强度 60–120 分钟或更长、题9 高强度 60 分钟以上、题10 经常力竭、题11 酸痛剧烈，任一满足即提示「单次活动过量」；题9 按选项下标 2 判断（非得分），避免误判。

### 3. 关键文件路径

| 功能         | 路径 |
|--------------|------|
| 测评主流程   | `app/src/pages/assessment/assessment.vue` |
| 档案页       | `app/src/pages/profile/profile.vue` |
| 历史详情     | `app/src/pages/profile/detail.vue` |
| 档案与历史读写 | `app/src/lib/profile-utils.ts`（含 `updateOrAppendBodyTestHistory`） |
| 体质/身体题目与选项 | `app/src/lib/assessment-data.ts` |
| 身体测试算分 | `app/src/lib/body-test-scoring.ts` |
| 测评与档案文案 | `app/src/lib/assessment-copy.ts` |
| 时间格式     | `app/src/lib/format.ts` |

### 4. 部署与指引

- H5：GitHub 推送后 Vercel 自动构建，输出目录 `app`，Build 命令 `npm run build:h5`，Output `dist/build/h5`。
- 合伙人测评指引：`docs/合伙人测评指引.md`（首次使用、更新档案、查看历史、反馈建议）。

## 二、后续模块

- **饮食（食养）**：已实现食养页（参考 Lovable 截图与 React Diet 逻辑）。`app/src/pages/diet/diet.vue`：时辰当令、你的饮食概览（体质宜忌）、安全提示、好评食谱、今日三餐、拍一拍识宜忌、食谱详情弹层、餐后感受反馈。数据与工具：`app/src/lib/diet-data.ts`（三餐示例、九种体质饮食宜忌、十二时辰）、`app/src/lib/diet-utils.ts`（getDietGuide、今日完成、餐后感受、好评食谱）；存储键 `benyuan_diet_completed`、`benyuan_diet_feelings`，同步 `benyuan_logs`。参考文档 `docs/reference/食养模块-Lovable参考.md`。
- **炼体**：首页/炼体页已读 profile，可展示身体测试推荐炼体法（活身法/动身法/合身法）及过量提示；待扩展具体炼体内容与交互。
