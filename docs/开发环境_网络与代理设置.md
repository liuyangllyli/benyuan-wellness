# 开发环境：网络与代理设置

开发时若使用**代理/VPN 访问外网**，浏览器可能把对 `localhost` 的请求也走代理，导致 **ERR_CONNECTION_REFUSED**（连接被拒绝）。  
解决办法：**让 localhost / 127.0.0.1 不走代理**。

---

## 一、推荐做法：代理绕过列表

在「不使用代理的地址」里加入（不同软件写法可能略有差异，任选其一或都写）：

- `localhost`
- `127.0.0.1`
- `::1`

这样访问本机开发服务器（如 `http://localhost:5173`）时不会经过代理，外网仍走代理。

---

## 二、常见设置位置（Windows）

### 1. 系统代理（影响大部分应用）

1. **设置** → **网络和 Internet** → **代理**
2. 在 **手动设置代理** 里找到 **「请勿将代理服务器用于本地(Intranet)地址」**，勾选
3. 或在 **编辑代理服务器列表 / 绕过列表** 中手动添加：`localhost;127.0.0.1`

### 2. 仅浏览器（Chrome / Edge）

- 若使用**系统代理**：同上，在系统里设置绕过即可
- 若使用**浏览器扩展**（如 SwitchyOmega、Proxy 等）：
  - 在扩展的「规则列表」或「直连 / 不使用代理」列表中加入：`localhost`、`127.0.0.1`
  - 或新建规则：条件为「主机名包含 localhost 或 127.0.0.1」，动作为「直连」

### 3. 环境变量（可选，给 Node/脚本用）

若某些脚本或工具会读系统代理，可设：

- 变量名：`NO_PROXY` 或 `no_proxy`
- 值：`localhost,127.0.0.1`

（开发前端页面时，主要保证**浏览器**访问 localhost 不走代理即可。）

---

## 三、项目内已做的配置

- **Vite（app）**：`app/vite.config.ts` 中已设置 `server.host: true`，便于本机与局域网访问；端口 5173，`strictPort: true`。
- 本地开发地址：在项目根目录执行 `cd app && npm run dev:h5` 后，用终端里打印的 **Local** 地址（如 `http://localhost:5173`）在浏览器打开。

---

## 四、快速自检

- 关闭代理后能打开 localhost → 说明是代理导致，按上面把 localhost 加入绕过即可。
- 关闭代理仍无法打开 → 先确认开发服务器已启动（终端有 `Local: http://localhost:5173`），再检查防火墙或杀毒软件是否拦截端口。
