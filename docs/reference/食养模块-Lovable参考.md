# 食养模块 · Lovable 参考（UI 与逻辑）

**用途**：实现 uni-app 食养页时对照 Lovable 的界面与数据流，仅作参考。

---

## 一、页面结构（对应截图）

1. **顶部**：返回、标题「食养之道」、标签「气·食养调气」；下方区块「戌时·心包经当令」「宜散步放松,愉悦心情」。
2. **你的饮食概览**：卡片内体质标签（如气虚质）、饮食方向（如益气健脾,温补为主）、「适合你的食材」绿色标签、「少吃」红色标签。
3. **安全提示**：食谱建议仅供参考，过敏体质请注意食材筛选。
4. **今日三餐**：早/午/晚三张卡片，每张有时段、标题、功效（如温补脾胃）；左侧圆点状态（未做/当前/已完成）；点击进入详情。
5. **拍一拍识宜忌**：AI 扩展卡片，拍照识食材、宜忌速查（多模态 AI 规划能力）。
6. **详情弹层（从三餐卡片进入）**：菜名、功效、食材（标签）、步骤（编号列表）、辰时/午时/酉时 TCM 建议、按钮「已完成这一餐」「换」。
7. **餐后反馈弹层**：完成一餐后弹出「这一餐感觉如何？」不适/一般/舒适/满足；记录感受后写入反馈，好评可进入「你觉得舒适的餐食」。

---

## 二、数据与存储（Lovable React 侧）

- **饮食建议**：由 `getDietGuideFromProfile(profile.constitutionType)` 得到 `{ type, dir, dos[], donts[] }`。
- **今日完成**：`getTodaySchedule()` 中 `type === 'diet'` 的已完成项；`markComplete(id)` 写入。
- **餐后感受**：`benyuan_diet_feelings` 数组 `{ mealId, feeling, timestamp }`；`benyuan_logs` 同步一条供归元趋势。
- **好评食谱**：`getTopRatedRecipes()` 基于饮食反馈统计，展示「你觉得舒适的餐食」。

---

## 三、静态数据（Lovable 示例）

- **三餐**：breakfast 山药红枣小米粥 / lunch 番茄扇贝豆腐汤 / dinner 莲子百合南瓜蒸；每餐含 name、benefit、ingredients、steps、tcmAdvice。
- **感受档位**：不适(1)、一般(2)、舒适(3)、满足(4)；对应反馈文案与「好评」逻辑（3、4 计入好评）。
