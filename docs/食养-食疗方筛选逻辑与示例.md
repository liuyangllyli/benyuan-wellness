# 食养·食谱分类与食疗方「能否吃」筛选逻辑

## 一、结论概览

- **早餐**、**家常菜**：**所有用户、任何体质**均可使用**全部**早餐食谱和**全部**家常菜食谱，不做体质筛选。
- **食疗方**：按体质规则筛选；**测评结果包含平和质**（是平和质或基本是平和质）时，可吃**全部**食疗方；**测评结果不包含平和质**时，仅展示/推荐「适合体质」**完全包含**用户体质集合的食疗方（用户体质集合 ⊆ 食谱适合体质列表）。

---

## 二、三类食谱与体质关系

| 类别     | 体质是否参与筛选 | 说明 |
|----------|------------------|------|
| 早餐     | 否               | 任何体质均可使用全部早餐食谱。 |
| 家常菜   | 否               | 任何体质均可使用全部家常菜食谱。 |
| 食疗方   | 是               | 见下节。 |

---

## 三、食疗方筛选逻辑（逐步说明）

### 1. 用户体质在系统里怎么存

- 测评结果会得到：
  - **类型** `constitution.type`：如「平和质」「阳虚质」「基本是平和质」
  - **倾向** `constitution.tendency`：如 `['阴虚质','血瘀质','气郁质']`
- 测评保存时：若主体质为「是平和质」或「基本是平和质」→ 存 **`type = '平和质'`**；否则存为对应偏颇体质（如 `'阳虚质'`）。倾向体质写入 `tendency` 数组。

### 2. 用于食疗方筛选的「用户体质集合」

- **代码**：`app/src/lib/diet-mock.ts` → `getShiliaoUserSet(profile)`
- **逻辑**：
  - 若 `profile.constitution.type === '平和质'`（即测评结果包含平和质）→ 返回 **`null`**，表示「全部食疗方可用」，不做体质过滤。
  - 否则（偏颇体质用户）：**用户体质集合** = `[type, ...(tendency || [])]`，即「是」的体质 + 所有「倾向」体质。不区分主体质/倾向，全部参与筛选。

### 3. 单条食疗方「能否吃」的判定（偏颇用户）

- **代码**：`app/src/lib/recipe-types.ts` → `recipeSuitsConstitutionSet(recipe, userConstitutionSet)`
- **数据**：食谱的「适合体质」列（`recipe.suitConstitution`）解析为集合（`parseSuitConstitution`，其中「普通正常质」视为「平和质」）。
- **判定**：当且仅当 **用户体质集合 ⊆ 食谱适合体质列表** 时，该食疗方**可用**（会展示、会推荐）。即食谱的「适合体质」必须包含用户集合中的**每一项**；食谱多写其他体质也可以。

### 4. 示例（偏颇用户）

- 用户：阳虚质（是）+ 倾向 阴虚质、血瘀质、气郁质 → 用户集合 = `['阳虚质','阴虚质','血瘀质','气郁质']`。
- 食谱 A：适合体质含「阳虚质、阴虚质、血瘀质、气郁质、气虚质」→ 包含上述四项 → **可用**。
- 食谱 B：适合体质仅「阳虚质、气虚质」→ 不包含阴虚质、血瘀质、气郁质 → **不可用**。

### 5. 小结

- **平和质用户**（`getShiliaoUserSet` 返回 `null`）：食疗方**全部**展示、全部可推荐。
- **偏颇用户**：食疗方仅当「适合体质」**完全包含**其体质集合（type + tendency）时才展示与推荐。
- **早餐、家常菜**：不按体质筛，所有用户均为全部早餐 + 全部家常菜。

## 四、实现与数据流

- **食谱库 / 共 N 道**：`fullRecipeList` = 全部早餐 + 全部家常菜 + 按上述规则筛选后的食疗方。  
  - 早餐：`getAllBreakfastRecipes()`  
  - 家常菜：`getAllJiachangRecipes()`  
  - 食疗方：`getShiliaoRecipesForProfile(getShiliaoUserSet(getProfile()))`
- **推荐**：`getSingleDayRecommendation`、`getWeekRecommendation` 均传入第五/第三参 `getShiliaoUserSet(getProfile())`，午/晚餐食疗方按同一规则从 `getRecipesByConstitutionDishTypeAndSlot(..., shiliaoUserSet)` 中选取。
- **代码位置**：  
  - `diet-mock.ts`：`getShiliaoUserSet`  
  - `recipe-types.ts`：`recipeSuitsConstitutionSet`、`parseSuitConstitution`  
  - `recipe-store.ts`：`getShiliaoRecipesForProfile`、`getRecipesByConstitutionDishTypeAndSlot`
