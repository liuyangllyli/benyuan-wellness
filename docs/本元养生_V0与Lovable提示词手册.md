# 本元养生 · V0 / Lovable 提示词手册（修订定稿版）

> **用途**：用于 V0 / Lovable 生成 UI 原型  
> **定位**：视觉与交互确定性，不是工程实现  
> **方法**：一页一提示，一天式用户旅程  

本文档已融合产品架构共识 + 展示类信息/完成感/弱化页面说明，便于一页一页稳定生成。

---

## 使用方式（请严格遵守）

**每次给 V0 / Lovable：**

1. **先完整粘贴【固定前言 Prompt】**（世界观 + 设计原则 + 禁区）
2. **再粘贴【当前页面 Prompt】**（一次只做一页）
3. **同时附上参考材料**
   - 去年 UI / 网页截图（视觉与信息层级参考）
   - Excel 用户旅程表或 PDF（一天 7 个时段的逻辑）

> ⚠️ **一次只画一页**；不要一次让模型画 5 页。首页满意后，再逐页往下做。

# 一、固定前言 Prompt（每次必贴）

> ⛔ 不可删减 · 每开一个新对话 / 新页面都要贴

```markdown
你正在为一个健康类小程序 / App 设计界面。

⚠️ 非常重要：这是一个【以“一天的生活节律”为核心】的养生产品，不是功能堆叠型健康 App。

---

## 产品名称
本元养生（BenYuan Wellness）

---

## 产品只回答一个问题
**「此刻，我最适合为身体做的一件事是什么？」**

---

## 核心设计原则（必须严格遵守）

1. **时间感优先**
   - 每一个页面，都必须能回答：
     - 现在是一天中的哪一刻？
     - 此刻 / 接下来 / 今天发生什么？
   - 通过：当前时段、时辰、今日进度、下一提醒来体现。

2. **首页不是功能集合**
   - 首页不是图标网格
   - 而是：**此刻 → 接下来 → 今天**

3. **一屏一事**
   - 每一屏只引导用户做“一件事”
   - 不并列多个主 CTA

4. **减法交互**
   - 不做复杂表格
   - 不做数据面板
   - 不让用户“管理健康”

5. **语气与气质**
   - 温和、克制、中式、陪伴感
   - 不制造焦虑
   - 不营销、不鸡血

6. **节律提醒**
   - 一天按 7 个时段：
     早餐 / 上午 / 午餐 / 午休 / 下午 / 晚餐 / 睡前
   - 页面文案与当前时段一致

---

## 页面关系（重要）

- **首页 = 一天的主界面**
  用户一天会多次回来

- **测评**：用户了解自身体质/状态后，**测评结果会直接指导后续的炼体推荐与饮食推荐**，因此测评页内需展示结果及对炼体、饮食的指导要点。

- **炼体 / 饮食 / 反馈**
  都是“嵌在时间里的页面”，
  不是平级功能模块；炼体与饮食的推荐依据来自测评结果。

---

## 展示类信息（不是独立功能页）

系统中存在一类【展示类信息】，用于“觉察”，不是操作入口：

- **节律与日期信息**
  如：当日节律、时辰、宜 / 不宜（弱提示即可）
- **情志信息**
  每日心语、情绪提醒
- **习惯提示**
  行 / 住 / 坐 / 卧中的不良习惯提醒

这些信息通常：
- 出现在首页顶部或底部
- 或作为轻提示嵌在页面中
- 不要求用户操作

---

## 完成感与打卡（重要）

- 用户每天的饮食 / 炼体是“有完成状态的”
- 不做复杂打卡系统
- 但必须让用户感觉：
  **“这一餐 / 这一练，今天已经完成了”**

---

## 系统中存在但弱化的页面（本次不必绘制）

- 设置 / 偏好（起居时间、提醒开关）
- 打卡汇总
- 统计 / 分析（未来会极简）

👉 当前页面中只需**预留延展感或轻入口**，不用设计完整页面。

---

## 设计禁区（请避免）

- 健身 App 风格
- 医疗 / 数据监控风格
- 工具 / 管理后台风格
- 多 CTA 并列
- 强营销感

---

## 参考材料说明
我会同时提供：① 去年网页/UI 截图；② Excel 用户旅程表或 PDF。请结合参考理解「一天式」的节奏与布局，但以本 Prompt 的页面要求为准。

---

## 技术说明（仅供理解）

- 最终用 uni-app 实现，用于微信小程序 + 独立 App
- 你只需输出：**页面结构、布局、示例文案**
```

## ② 本元测评中心

```markdown
现在请你设计【本元测评中心】页面。

### 页面角色
- 用户主动来“了解身体”，测评结果将**指导后续的炼体推荐与饮食推荐**
- 可能是首次，也可能是复测
- 本页包含两个阶段：**答题过程** + **测评结果展示**

---

### 阶段一：答题过程

1. 测评进度（如：第 2 / 6 题）
2. 当前题目（单题展示）
3. 选项（点击或滑动）
4. 上一题 / 下一题
5. 安全 / 禁忌提示（非常克制）

设计重点：一题一屏；像对话，不像问卷；不出现复杂解释。

---

### 阶段二：测评结果展示（必做，与炼体/饮食衔接）

答题完成后，**必须在测评页内展示测评结果**，并明确说明这些结果会对炼体和饮食产生指导作用。结构要求：

1. **结果总览**
   - 体质类型 / 身体状态概要（如：平和质、阳虚质等，或简要描述）
   - 可选：疼痛或受限部位提示（极简，不制造焦虑）

2. **对炼体的指导（在本页展示）**
   - 一段简短文案：根据当前体质/状态，炼体推荐的方向或原则（如：宜舒缓开肩、不宜剧烈跑跳）
   - 可配 1–2 条要点或标签，让用户知道“后面的炼体内容会按这个来”

3. **对饮食的指导（在本页展示）**
   - 一段简短文案：根据当前体质/状态，饮食推荐的方向或原则（如：宜温养脾胃、少生冷）
   - 可配 1–2 条宜忌要点，让用户知道“后面的三餐推荐会按这个来”

4. **衔接说明（轻量）**
   - 一句提示：如“以上结果将用于为您推荐每日炼体与饮食”，或“返回首页即可获得个性化推荐”
   - 主 CTA：返回今日中心 / 开始体验

---

### 整体要求
- 结果页信息量克制，不展开成“报告”
- 炼体指导、饮食指导在同一屏或可滑动的一页内呈现，不拆成多个独立报告页
- 语气温和，不贴标签、不制造身体焦虑

请输出：① 答题过程页面结构 + ② 测评结果展示页面结构（含炼体指导、饮食指导区块）+ 示例文案。
```

### Q1：底部导航要不要 6 个 tab（首页/测评/饮食/运动/打卡/分析）？

**标准回答（可复制给 BOLT）：**

本产品是「一天式节律」为主，首页是主界面，不是功能平铺。

- **不要** 6 个平级 tab；**不要**把「打卡」「分析」作为底部一级入口。
- **可以**二选一：
  - **方案 A**：底部只保留「首页」一个主入口，测评/炼体/饮食/反馈都从首页在「具体时间点」或「此刻推荐」进入。
  - **方案 B**：底部保留 3–4 个入口，例如「首页」「测评」「饮食」「炼体」（首页仍为主，其余为快捷入口），不设「打卡」「分析」tab。

请按你当前生成的页面选择其一，并保持首页为「此刻 → 接下来 → 今天」的主线，不要做成图标网格。

---

### Q2：情志是否单独做一个轻量页面？

**标准回答：**

情志 / 日期 / 习惯属于「展示类信息」，不是独立功能页。

- **不要**为情志单独设计一整页（避免变成情绪管理、选择、解释）。
- **只作为**：首页的轻提示（如今日心语）、或嵌在炼体/饮食页的一行/小块文案。

---

### Q3：测评从哪里进？测评完成后去哪里？

**标准回答：**

- **入口**：测评**只**从首页的「更精准（去测评）」进入，不做独立测评 tab、不强制新用户先测评。
- **完成后**：在测评页内展示「结果 + 对炼体/饮食的指导」，然后主 CTA 为「返回今日中心」或「开始今日推荐」。不做独立报告页，也不要无提示直接跳回首页。

---

### Q4：配色用渐变色系还是单一主色？

**标准回答：**

可参考附带的 PDF/截图中的低饱和渐变色系，但**一页一主调**，不在同一屏堆多种高饱和色。

建议：首页偏中性（米白/浅绿），炼体偏静（青/蓝），饮食偏暖（橙/米），测评偏中性（浅灰/淡紫）。整体安静、低对比、不抢注意力。

---

### Q5：是否一次生成 5 个页面？

**标准回答：**

产品共有 5 个核心页面（首页、测评、炼体、饮食、反馈），但**本次只生成我指定的这一页**，其他页面仅作上下文理解，不要一次输出多页。生成顺序建议：首页 → 测评 → 炼体 → 饮食 → 反馈。
