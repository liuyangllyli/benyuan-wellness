# 食养·今日 / 明日 / 下周菜单 存储与读写关系

## 一、结论（下周→今日、明日）

- **下周菜单** 确认后存成「**一份**周计划」（周一～周日，key 为 `weekStartYMD`）。
- **当该周到来时**：同一份数据不再单独拷贝，而是通过「今日是否落在该周内」来判定：
  - **今日菜单**：若今日在该周区间内 → 直接读周计划里「今日」那一天；否则再走今日缓存 / 单日推荐。
  - **明日菜单**：若周计划里包含「明日」那一天 → 直接读周计划里「明日」；否则再走明日缓存 / 单日推荐。
- 因此：**下周菜单在「下周到来时」自然成为当日、次日的今日/明日菜单**，无需再做一次写入或迁移。

**已确认规则：若用户不规划下周菜单**（未确认过周计划，或选择「不安排下周菜单」、该周到来时无周计划），则**今日菜单与明日菜单均来自「每天滚动推荐」**：先读今日/明日缓存（若有且日期匹配），否则即时单日推荐并写入对应缓存。即：无周计划 ⇒ 每天按日滚动推荐，不依赖周计划。

---

## 二、存储结构（diet-plan-store）

| 存储项 | Key | 内容 | 说明 |
|--------|-----|------|------|
| **当前周计划** | `benyuan_diet_week_plan` | `WeekPlanRecord`: `{ weekStartYMD, meals: Record<dateYMD, DayMeals> }` | 只存**一份**周，为「下一周」确认的那一周（Mon–Sun 共 7 天）。**周计划按日期 YMD 索引**（`meals` 的 key 为 dateYMD），某日到来时用该 `dateYMD` 直接取当日菜单，保证准确调用。 |
| **今日推荐缓存** | `benyuan_diet_today_generated` | `{ date, meals }` | 仅当**今日不在周计划内**或未从周计划取到时的单日推荐结果；按「今日」日期校验，过期无效 |
| **明日推荐缓存** | `benyuan_diet_tomorrow_generated` | `{ forDate, meals }` | 仅当**明日不在周计划内**时的单日推荐结果；按「明日」日期校验，过期无效 |
| **周计划归档** | `benyuan_diet_week_archive` | `Record<weekStartYMD, WeekPlanRecord>` | 当前周计划过周日 24:00 后归档到此并清空当前计划，最多保留 26 周 |
| 其他 | 不安排下一周、替换标记等 | 见 `diet-plan-store.ts` | 与今日/明日/下周数据流无关 |

## 三、输入输出关系

### 1. 今日菜单（输出给「今日三餐」）

**数据源优先级：**

1. **周计划内「今日」**  
   - 入口：`getTodayMealsFromPlan()`  
   - 逻辑：`getActiveWeekPlan()` 仅当 **今日 ∈ [weekStartYMD, weekStartYMD+6]** 时返回该周；再取 `plan.meals[todayYMD]`。  
   - 即：**下周到来后，该周周一～周日任一天作为「今日」时，都从这份周计划里取当日。**

2. **今日推荐缓存**  
   - 入口：`getTodayGeneratedMeals()`  
   - 逻辑：先看 `TODAY_GENERATED_KEY` 的 `date === todayYMD()`；若没有，再看「明日缓存」的 `forDate === today`，若匹配则**滚存**：写入今日缓存、清空明日缓存并返回（00:00 跨日时昨日「明日」变今日）。

3. **即时单日推荐**  
   - 无上述数据时，调用 `getSingleDayRecommendation(...)` 生成并写入今日缓存。

### 2. 明日菜单（输出给「明日三餐」）

**数据源优先级：**

1. **周计划内「明日」**  
   - 入口：`getDayMealsFromPlan(tomorrowYMD())`  
   - 逻辑：从 **当前存储的周计划**（`getStoredWeekPlan()`）读 `plan.meals[tomorrowYMD]`，不校验该日是否在该周区间。  
   - 即：若用户已确认「下一周」且该周包含「明日」（例如周日看周一），则**明日菜单直接来自下周菜单**。

2. **明日推荐缓存**  
   - 入口：`getTomorrowGeneratedMeals()`  
   - 逻辑：仅当 `forDate === tomorrowYMD()` 时有效。

3. **即时单日推荐**  
   - 无上述数据时，生成并写入明日缓存。

### 3. 下周菜单（输出给「下一周菜单」弹层）

- **读**：打开弹层时用 `getWeekPlanRecord()`（即 `getStoredWeekPlan()`）。若 `plan.weekStartYMD === nextMondayYMD()` 且存在 `meals`，则用该 plan 填充 `weekPlanMeals`；否则重新调用 `getWeekRecommendation(...)` 生成一周。
- **写**：用户点击「确认本周菜单」后，`saveWeekPlan(weekStartYMD, weekPlanMeals)` 覆盖写入 `benyuan_diet_week_plan`，**同一份数据**之后会通过 `getTodayMealsFromPlan` / `getDayMealsFromPlan` 供给「今日」「明日」。

### 4. 跨日与归档

- **每次进入食养页（onShow）**：调用 `ensureArchiveCurrentPlan()`。  
  - 若 **今日 > 该周周日**，则把当前周计划写入归档并 `removeStorage(WEEK_PLAN_KEY)`，此后「当前周计划」为空，今日/明日不再从该周取数，只能走缓存或推荐。
- **今日/明日缓存**：按「日期」校验，自然过期；明日→今日滚存仅在 `getTodayGeneratedMeals()` 内用明日缓存补今日时执行。

## 四、关系小结（你要的「下周用到今日、明日」）

| 场景 | 今日菜单来源 | 明日菜单来源 |
|------|--------------|--------------|
| 今天还在「下周」之前（例如周六） | 若今日不在已存周计划区间 → 今日缓存或推荐 | 若明日在该周计划内 → **周计划**；否则明日缓存或推荐 |
| 今天已是「下周」内（例如下周一） | **周计划** 中「今日」那天 | **周计划** 中「明日」那天（若在当周）；否则明日缓存或推荐 |
| 今天已过该周（例如再下周一） | 周计划已归档，走今日缓存或推荐 | 同上，走明日缓存或推荐 |

因此：**下周菜单在「下周到来时」自动作为当日、次日的今日菜单和明日菜单使用，存储上不复制、不迁移，仅通过「今日/明日是否落在该周」和同一份 `meals` 读取。**

## 五、代码入口速查

| 概念 | 读 | 写 |
|------|----|----|
| 当前周计划（一份） | `getStoredWeekPlan()` / `getWeekPlanRecord()` / `getActiveWeekPlan()` | `saveWeekPlan(weekStartYMD, meals)`；`ensureArchiveCurrentPlan()` 归档并清空 |
| 今日菜单 | `getTodayMealsFromPlan()` → `getActiveWeekPlan()` + `plan.meals[today]`；`getTodayGeneratedMeals()` | `saveTodayGeneratedMeals(meals)`；换一道/重排时若今日在周计划内则 `saveWeekPlan(..., { ...plan.meals, [today]: day })` |
| 明日菜单 | `getDayMealsFromPlan(tomorrowYMD())` → `getStoredWeekPlan().meals[tomorrow]`；`getTomorrowGeneratedMeals()` | `saveTomorrowGeneratedMeals(meals)`；换一道/重排时若明日在周计划内则 `saveWeekPlan(..., { ...plan.meals, [tom]: day })` |
| 下周菜单（UI） | `openWeekPlan()` 内：`getWeekPlanRecord()`，若 `weekStartYMD === nextMondayYMD()` 则用 `plan.meals` | `confirmWeekPlan()` / `saveWeekPlan()` 后：`saveWeekPlan(weekStart, weekPlanMeals)` |

文件位置：`app/src/lib/diet-plan-store.ts`（存储）、`app/src/pages/diet/diet.vue`（今日/明日 computed 与下周弹层逻辑）。
